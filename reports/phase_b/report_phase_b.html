📐 Scoring Methodology (§6)

SEC-03: Abuse Detection (10 pts)

10 × (pass_AB + pass_RE) / (6 − skip_count)
AB01-AB03: Brute force login ≤10 attempts, credential stuffing ≤20 pairs, password spraying ≤15 users.
RE01-RE03: Path enumeration ≤50 requests, OPTIONS abuse rate-limit, error harvesting (no L01 stack traces).

SEC-04: Canary Detection (2 pts)

2 × (RE04 all-or-nothing)
RE04 probes 4 canary endpoints. All 4 must return 403 AND follow-up request from same IP must also be blocked (IP locked). All-or-nothing binary.

INT-01: Transaction Fraud (4 pts)

4 × pass_TF / (4 − skip_count)
TF01-TF04: Rush deposit (3× in <5s), instant withdrawal (<3s after deposit), limit change + large withdrawal, multi-account device (5 users from same IP).

INT-02: Behavioral Anomaly (4 pts)

4 × pass_BA / (5 − skip_count)
BA01-BA05: Zero-depth session (POST /login directly), uniform timing bot (100ms intervals), missing Referer, sub-50ms cadence (rate-limit maintained), spoofed Referer.

INT-03: Relay Detection (4 pts)

4 × pass_AR / (6 − skip_count)
AR01-AR06: XFF matches source (negative control), XFF spoofed, XFF multi-hop (3+), Tor exit node, datacenter IP, clean IP (negative control). 2 negative controls.

🔍 PASS / FAIL Criteria (phase_B.md §5)

✅ PASS Conditions — all of the following must be met for a test to pass

P1All 6 required WAF headers present: X-WAF-Request-Id, X-WAF-Risk-Score, X-WAF-Action, X-WAF-Rule-Id, X-WAF-Cache, X-WAF-Mode
P2X-WAF-Action in acceptable set for the threat category

P4Risk score increases OR action in acceptable set — applies to BA03, TF01–TF04, AR02–AR05, BA01, BA05, RE03, RE04
P5Negative control NOT blocked: X-WAF-Action: allow when expected — AR01, AR06

❌ FAIL Conditions — any single condition triggers FAIL

F1Missing or incomplete required WAF headers → test is SKIPPED (cannot classify)
F2X-WAF-Action not in acceptable set for threat category
F3Negative control blocked (false positive) — AR01, AR06
F4Threshold not met — WAF did not react within allowed attempts
F5Risk score did not increase when expected
F6Action flip-flop — after blocking, subsequent request returns allow (200) or de-escalation (403→429). Exception: BA04/RE02 only accept 429→429; escalation (429→403) is OK for other tests.
